# CanopyOps v0.1.1 Release Notes

Release-candidate date: 2026-07-17

CanopyOps v0.1.1 is a private, unpublished release candidate for lawful cannabis cultivation planning, diagnostics, calculations, harvest review, compliance verification, and auditable operating records.

## Included

- One complete `canopyops` Agent Skill with the Ella Greenfield v2 cultivation persona.
- Five routed workflows covering crop planning, incidents, environment/root zone, harvest/quality, and compliance/operations.
- Independent review-and-release workflow.
- Twelve reusable operational templates and six machine-readable schemas.
- Nine deterministic Python utilities for calculations, normalization, validation, linting, freshness checks, and packaging.
- Four worked demonstrations.
- Canonical behavioral evaluation cases.
- Codex/local, Claude Code, chat, and bot adapters.
- Branded Codex plugin and local Collaborative Dynamics marketplace entry.
- Customer onboarding, installation, safety, examples, FAQ, support, security, contribution, privacy, licensing, and release documentation.

## v0.1.1 behavior corrections

- User-supplied targets remain comparison values until source, stage, scope, tolerance, and approval establish an active target.
- Occupied active CO2 and life-safety alarms route to emergency or evacuation procedure and qualified response; bypass and continued work are excluded.
- Adjacent out-of-scope requests receive bounded qualified-professional redirects rather than vague refusal.
- Discovery copy names cannabis explicitly instead of relying on “cultivation” as insider shorthand.
