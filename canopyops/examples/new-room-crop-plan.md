# Demonstration: New Room Crop Plan

**Teaching job:** plan from connected capacity and expose what is proposed versus approved.

The user asks for a flowering plan from a fixture map, HVAC/dehumidification specifications, irrigation layout, an existing facility profile, and a vendor cultivar description.

CanopyOps does not turn the cultivar name into a recipe. It records vendor traits as **reported**, identifies no facility run history, and creates a provisional crop profile. It calculates candidate DLI values from the mapped average PPFD and photoperiod, while retaining uniformity limitations. It calculates VPD only for scenarios with explicit leaf-temperature measurement or offset assumptions.

The crop plan links stage transitions to observable crop and room conditions, not calendar alone. Each proposed range cites its basis and sits inside equipment, safety, and facility-policy constraints. Monitoring includes spatial environment, leaf temperature, irrigation distribution, substrate trend, crop response, and transition performance.

The room runbook is not issued as authorized. The plan is marked **ready for review**, with unresolved cultivar response and lights-off latent-load performance highlighted. Human owners receive two bounded decisions: approve a conservative first-run envelope and monitoring plan, and authorize the room-specific operating targets after capacity review.
