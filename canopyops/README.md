# CanopyOps

CanopyOps equips an agent to turn cannabis cultivation observations, logs, constraints, and goals into legal-market crop plans, incident workups, environmental and root-zone calculations, harvest decisions, and auditable operating records.

It is an advisory work system. It does not operate controllers, authorize pesticides, interpret law, release batches, replace laboratory testing, or provide medical, extraction, covert-cultivation, or enforcement-evasion guidance.

## Start

Install this directory as the `canopyops` skill, then ask naturally:

- “Build a crop plan for this room and cultivar.”
- “Help diagnose rising runoff EC.”
- “Turn last night’s humidity excursion into an incident record and next actions.”
- “Review this batch for harvest readiness.”
- “Prepare a compliance verification brief for this proposed change.”

CanopyOps will inspect supplied files before interviewing you, state its provisional reading, and ask only for facts that change the next decision. Markdown, CSV, and JSON are first-class outputs. Python scripts are optional deterministic aids.

## Claude Code

Install the complete `canopyops` directory at `~/.claude/skills/canopyops/` for personal use or `.claude/skills/canopyops/` inside one project. Claude Code can then select it automatically from the description or run it directly with `/canopyops`. Keep every supporting directory beside `SKILL.md`; see `adapters/claude-code.md` for host behavior.

## Operating status

Outputs distinguish observation, assumption, calculation, interpretation, recommendation, approval, execution, and verification. A recommendation is never an authorization. Facility targets, labels, SOPs, current jurisdiction sources, and accountable humans remain authoritative for consequential action.
