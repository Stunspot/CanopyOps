# CanopyOps Data and Privacy

CanopyOps v0.1.4 is distributed as a local Codex plugin, a standalone Agent Skill, and a Claude.ai upload archive. These distributions include no account, telemetry, analytics, hosted service, connector, MCP server, hook, or automatic network request. Collaborative Dynamics does not receive cultivation records, facility data, prompts, or generated outputs through CanopyOps itself.

Data entered while using CanopyOps is handled by the Codex host and any model or tools the user chooses to invoke. Their terms, privacy controls, retention rules, and network behavior govern that processing. Users remain responsible for deciding whether their host and selected tools are appropriate for confidential facility, employee, customer, compliance, or trade-secret information.

CanopyOps can write local artifacts when the user asks it to. Those files remain wherever the user or host places them. The plugin does not upload, synchronize, delete, or transmit them by itself.

This statement describes the public v0.1.4 package as built. Any future connector, hosted service, telemetry, or tool-backed edition requires a new privacy review and an updated statement before release.
